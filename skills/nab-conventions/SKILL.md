# nab conventions

Static HTML/CSS/JS site generated by a Node build. Dev: `pnpm dev:nab` builds then serves on port 8080 (`PORT` to override). No framework, no linter, no unit tests.

## The one rule that matters most

The 70+ `*.html` files in `apps/nab/` are **generated output**. Never edit them directly. The sources of truth are:

- `scripts/pages.js` — page definitions (content, sections, nav).
- `scripts/sections.js` — section renderers.
- `templates/page.html`, `templates/header.html`, `templates/footer.html` — shells.
- `css/styles.css`, `js/main.js`, `js/datalayer.js` — static assets served as-is.

Workflow: edit sources → `npm run build` (or `pnpm dev:nab`) → verify output. `prebuild` runs `scripts/sync-tokens.js`, which copies `@demo/ui/tokens.css` into `css/tokens.css` — edit the package, never the copy.

## Styling

- NAB tokens in `css/styles.css :root`: `--nab-red: #c80000`, `--nab-red-bright: #ed0000`, `--nab-red-dark: #a50000`, `--ink: #1a1a1a`, `--bg-tint: #f7f5f2`, `--radius: 12px`, `--radius-lg: 20px`, `--container: 1200px`, `--header-h: 72px`.
- Font stack: `"Helvetica Neue", Helvetica, Arial, "Segoe UI", Roboto, sans-serif`.
- BEM-ish class naming: `.btn`, `.btn--primary`, `.accordion__btn`, `.main-nav__btn`, `.link-card__more`.
- Mobile menu breakpoint: `880px`. Animations are plain CSS transitions.

## Adobe Client Data Layer (ACDL)

`js/datalayer.js` vendors `@adobe/adobe-client-data-layer` and exposes `window.NabDataLayer = { pushPageLoad, pushClick }`. The page template bootstraps `window.adobeDataLayer = window.adobeDataLayer || []` before any script.

- Page loads push `cmp:show` with `page` metadata (`@type: "nab/components/page"`, `dc:title`, `xdm:language`, `repo:path`, url, referrer, timestamp).
- Clicks push `cmp:click` with `eventInfo.path = "component.<id>"` and a `component` object.
- `js/main.js` wires triggers: mega-menu opens, mobile menu, search, login, accordions, and a global click delegation on `[data-dl-id], .btn, .link-card__more, .article-card__link`.

**When adding or changing interactive elements, keep ACDL working:** give new clickable elements a `data-dl-id` attribute (or one of the delegated classes) so clicks are tracked. Verify in the browser console — dev logging prints `[adobeDataLayer]` entries via `console.debug` on page load and clicks.

## Accessibility (nab is the strictest app)

- Keep the skip link (`.skip-link`).
- Mega-menu: `aria-expanded` + `aria-controls` on triggers, `hidden` on closed panels.
- Mobile menu: `role="dialog"`, `aria-modal="true"`, `aria-hidden` on the overlay.
- Breadcrumbs: `aria-label="Breadcrumb"` and `aria-current="page"`.
- The home page accessibility accordion is injected by `scripts/build.js` with proper `aria-expanded`/`aria-controls` — preserve that behavior when touching the build.

## Testing caveat

nab's `test` script is a Playwright walkthrough recorder, not a unit suite — it requires `pnpm exec playwright install chromium` and is why root `pnpm test` fails. Treat it as optional; never "fix" root test by removing it.
